# vba-implement｜VBA実装エージェントワークフロー

「〇〇するツールを作って」という依頼を受けたとき、このファイルの手順に従って仕様から .cls ファイルまで一気通貫で進める。

---

## フロー概要

```
要件ヒアリング → 仕様確定 → コード生成 → 自己レビュー → ファイル出力
```

各ステップで必ずユーザーの確認を取ってから次へ進む。

---

## ステップ1：要件整理

`.agent/skills/vba-spec/SKILL.md` の手順に従い、要件を構造化して `.spec/SPEC.md` に書き込む。
**ユーザーの確認が取れてからステップ2へ進む。**

---

## ステップ2：モジュール設計

`.agent/skills/vba-new-module/SKILL.md` のステップ1〜2（仕様確認・関数名決定）を実行する。
**関数名一覧をユーザーに提示して確認が取れてからステップ3へ進む。**

---

## ステップ3：コード生成

`.agent/skills/vba-new-module/SKILL.md` のステップ3〜5（骨格生成・注意事項チェック・品質確認）を実行する。
コードはチャットに出力せず、プロジェクトフォルダ直下に `.cls` ファイルとして書き出す。
**「[モジュール名].cls を生成しました」と報告してステップ4へ進む。**

---

## ステップ4：自己レビュー

`.agent/skills/vba-review/SKILL.md` の手順に従い、生成したファイルをレビューする。
NG箇所があれば自己修正してから「レビュー完了」を報告する。

---

## ステップ5：完了報告

以下を報告して終了する。

- 生成したファイルパス
- 公開マクロ名と使い方
- 次にやること（SPEC.mdのTODO更新など）
