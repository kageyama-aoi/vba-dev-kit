# vba-spec｜VBA要件整理ワークフロー

ユーザーから「〇〇したい」という雑多な説明を受けたとき、このファイルの手順に従って要件を整理すること。

---

## ステップ1：ヒアリング

以下の項目が不明な場合はユーザーに確認する。すべて揃ったらステップ2へ進む。

- **目的**：何のためのツールか（1〜2文）
- **入力**：何を入力とするか（ファイル・シート名・列構成など）
- **出力**：何を出力するか（書き込み先・形式・列構成など）
- **処理の流れ**：大まかなステップ（3〜5段階）
- **既存コードとの関係**：流用するモジュールがあるか

---

## ステップ2：構造化して出力

以下のフォーマットで整理して提示し、ユーザーに確認を求める。

```
### 目的
{1〜2文}

### 入出力
- 入力：{データ形式・ファイル種別・シート名・列構成}
- 出力：{書き込み先・形式・列構成}

### 処理フロー
1. {ステップ1}
2. {ステップ2}
3. {ステップ3}

### 必要機能
- {機能1}
- {機能2}

### 既存コードとの関係
- 流用：{あれば / なし}
- 新規作成：{Yes / No}

### 設計ルール（固定）
- Main は流れのみ・実処理は関数分割
- Excel書き込みは最後に一括（ループ内セルアクセス禁止）
- ScreenUpdating / Calculation / EnableEvents を停止・復元
- On Error GoTo Cleanup + errContext でエラー箇所を特定
- UTF-8ファイルは ADODB.Stream で読み込む
```

---

## ステップ3：.spec/SPEC.md に書き込む

ユーザーの確認が取れたら `.spec/SPEC.md` に書き込み、「SPEC.mdに保存しました」と報告する。
