# vba-new-module｜新規VBAモジュール作成ワークフロー

新しい .cls / .bas ファイルを作成するとき、このファイルの手順に従うこと。

---

## ステップ1：仕様の確認

`.spec/SPEC.md` が存在する場合はまず読む。
存在しない場合は `.agent/skills/vba-spec/SKILL.md` の手順で要件整理を先に行う。

---

## ステップ2：関数名の決定

`docs/関数命名ルール.md` を読み、以下を決定してユーザーに提示・確認する。

- モジュール名
- 公開マクロ名（Main相当）
- 主要な内部関数名（3〜5個程度）

確認が取れてからコード生成に進む。

---

## ステップ3：コード骨格の生成

`docs/VBAツール標準テンプレート.md` のテンプレートを使い、以下を含む骨格を生成する。

- モジュールヘッダーコメント
- `Option Explicit`
- `Private Const` による定数宣言
- `Main` サブルーチン（On Error GoTo Cleanup・パフォーマンス設定つき）
- 各関数のスタブ（関数コメント + 空実装）
- `Cleanup` ラベル

---

## ステップ4：注意事項チェック

`docs/VBA注意事項.md` の全項目を対照して、生成したコードが違反していないか確認する。
違反があれば自己修正してからユーザーに提示する。

---

## ステップ5：品質最低基準の確認

以下をすべて満たしているか確認してから提示すること。

- [ ] `Option Explicit` が先頭にある
- [ ] モジュール変数は `Private` で宣言されている
- [ ] ループ内でセルを直接操作していない
- [ ] ファイル読み込みは ADODB.Stream（UTF-8の場合）
- [ ] エラー時に `ScreenUpdating / Calculation / EnableEvents` が復元される
- [ ] 文字列リテラルの繰り返しは `Private Const` に抽出されている
- [ ] 関数名は「動詞 + 名詞」になっている
