# vba-review｜VBAコードレビューワークフロー

レビュー対象の .cls / .bas ファイルを受け取ったとき、このファイルの手順に従うこと。

---

## ステップ1：対象ファイルを読む

ユーザーが指定したファイルを読み込む。
指定がない場合はプロジェクト内の .cls / .bas ファイルを一覧して確認する。

---

## ステップ2：注意事項チェック（`docs/VBA注意事項.md` 対照）

セクション A は全コードが対象。B / C はコード内に該当処理がある場合のみチェックする。

### セクション A：モジュール骨格（常にチェック）

| 項目 | 結果 | 該当箇所 |
|------|------|---------|
| A-1：モジュールレベル宣言（Const/Dim）が最初の Sub より前にある | ✓/✗ | |
| A-2：モジュール変数・定数は Private で宣言されている | ✓/✗ | |
| A-3：Dim はサブルーチン先頭にまとめられている | ✓/✗ | |
| A-4：パフォーマンス設定の定型句がある（ScreenUpdating 等） | ✓/✗ | |
| A-5：errContext でエラー発生箇所を特定している | ✓/✗ | |

### セクション B：Excel操作パターン（Excel操作がある場合のみ）

| 項目 | 結果 | 該当箇所 |
|------|------|---------|
| B-1：ループ内でセルを直接操作していない | ✓/✗ | |
| B-2：UsedRange を使っていない（最終行は End(xlUp).Row） | ✓/✗ | |
| B-3：Variant配列が1始まりで正しく処理されている | ✓/✗ | |
| B-4：配列書き込みを Resize.Value で一括で行っている | ✓/✗ | |
| B-5：長い処理で StatusBar を使っている | ✓/✗ | |

### セクション C：コーディング落とし穴（該当する処理がある場合のみ）

| 項目 | 結果 | 該当箇所 |
|------|------|---------|
| C-1：ファイル読み込みは ADODB.Stream（UTF-8対応） | ✓/✗ | |
| C-2：ReDim Preserve する配列は動的宣言（Dim arr()） | ✓/✗ | |
| C-3：And/Or 条件内に配列アクセスを混在させていない | ✓/✗ | |
| C-4：Variant要素を ByRef String 引数に直接渡していない | ✓/✗ | |
| C-5：Dictionary へのオブジェクト格納に Set を付けている | ✓/✗ | |

---

## ステップ3：品質最低基準チェック

| 項目 | 結果 |
|------|------|
| `Option Explicit` が先頭にある | ✓/✗ |
| ループ内でセルを直接操作していない | ✓/✗ |
| エラー時に ScreenUpdating 等が復元される | ✓/✗ |
| 文字列リテラルの繰り返しは Private Const にある | ✓/✗ |
| 関数名は「動詞 + 名詞」になっている | ✓/✗ |

---

## ステップ4：結果報告

- ✗ の項目は **該当行数と修正案**をセットで提示する
- すべて ✓ の場合は「レビュー完了：問題なし」と報告する
- 修正するかどうかはユーザーに確認してから行う
