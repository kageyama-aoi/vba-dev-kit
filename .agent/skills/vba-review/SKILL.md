# vba-review｜VBAコードレビューワークフロー

レビュー対象の .cls / .bas ファイルを受け取ったとき、このファイルの手順に従うこと。

---

## ステップ1：対象ファイルを読む

ユーザーが指定したファイルを読み込む。
指定がない場合はプロジェクト内の .cls / .bas ファイルを一覧して確認する。

---

## ステップ2：注意事項チェック

`docs/VBA注意事項.md` の各項目を対照してチェックする。

| 項目 | 結果 | 該当箇所 |
|------|------|---------|
| UTF-8読み込みはADODB.Stream | ✓/✗ | |
| ループ内でセルを直接操作していない | ✓/✗ | |
| パフォーマンス設定の定型句がある | ✓/✗ | |
| errContextでエラー箇所を特定している | ✓/✗ | |
| Dimはサブルーチン先頭にまとめている | ✓/✗ | |
| 固定サイズ配列にReDimしていない | ✓/✗ | |
| UsedRangeを使っていない | ✓/✗ | |
| Variant配列が1始まりで正しく処理されている | ✓/✗ | |
| 配列書き込みを一括で行っている | ✓/✗ | |
| ByRef引数にVariant要素を渡していない | ✓/✗ | |
| モジュール変数はPrivateで宣言されている | ✓/✗ | |

---

## ステップ3：品質最低基準チェック

| 項目 | 結果 |
|------|------|
| `Option Explicit` が先頭にある | ✓/✗ |
| ループ内でセルを直接操作していない | ✓/✗ |
| エラー時にScreenUpdating等が復元される | ✓/✗ |
| 文字列リテラルの繰り返しはPrivate Constにある | ✓/✗ |
| 関数名は「動詞 + 名詞」になっている | ✓/✗ |

---

## ステップ4：結果報告

- ✗ の項目は **該当行数と修正案**をセットで提示する
- すべて ✓ の場合は「レビュー完了：問題なし」と報告する
- 修正するかどうかはユーザーに確認してから行う
